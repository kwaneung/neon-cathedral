## 3. 선정 항목: 배경 및 문제 정의

### 3.1 배경

네온 성당의 차별화 포인트 중 하나는 **"태운 뒤 5분, 혼자 남는 성찰의 시간"**과 그 후 도착하는 **편지봉투**이다 (REQUIREMENTS §5, FR-5.2). 현재 서버는 고해 제출 시 `sent_at = created_at + 5분`으로 답장을 예약하고, `getUserReplies`가 `sent_at <= now` 조건으로 필터링한다.

그러나 클라이언트는 이 흐름을 사용자에게 전달하지 못한다.

### 3.2 현재 문제

1. **대기 상태 불가시** — 고해 제출 후 남은 시간·도착 예정 시각을 보여주지 않음
2. **수동 새로고침 의존** — 편지함 빈 상태 카피가 "5분 뒤 새로고침 버튼을 터치"를 안내 (```769:771:src/app/page.tsx```)
3. **도착 알림 부재** — 하단 네비 '서신' 탭에 미수신 편지 뱃지 없음
4. **폴링 비효율** — 15초 전역 폴링은 편지 대기 중에도 동일 주기로 동작하나, 도착 시점 피드백 없음
5. **브랜드 내러티브 단절** — "5분, 번뇌가 정화되는 시간" 카피(```752:755:src/app/page.tsx```)는 있으나 인터랙티브 경험으로 연결되지 않음

### 3.3 기대 효과

- 핵심 사용자 여정의 **감성적 완결성** 확보
- 재방문·이탈 없이 **자연스러운 편지 수령** 유도
- Server Actions·기존 스키마만으로 **빠른 MVP 품질 향상**

## 4. 상세 요구사항

### 4.1 사용자 스토리

| ID | 스토리 | 우선순위 |
|---|---|---|
| US-1 | 고해자로서, 고해를 태운 뒤 **답장이 도착하기까지 남은 시간**을 볼 수 있어 기다림이 불안하지 않았으면 한다 | Must |
| US-2 | 고해자로서, 5분이 지나 편지가 도착하면 **앱 어디에 있든 알림(뱃지·토스트)**을 받아 편지함으로 이동할 수 있었으면 한다 | Must |
| US-3 | 고해자로서, 편지함에 들어가면 **대기 중인 편지(봉인 상태)**와 **도착한 편지(개봉 가능)**를 구분해 볼 수 있었으면 한다 | Should |
| US-4 | 고해자로서, 이미 도착한 편지를 열람했음을 시스템이 기억해 **읽음/미읽음**을 구분했으면 한다 | Should |
| US-5 | 모바일 사용자로서, 성찰 대기 중에도 **배터리를 과도하게 소모하지 않는** 폴링 방식이었으면 한다 | Must |

### 4.2 동작 명세

#### 4.2.1 대기 중 편지 조회 (Server Action 신규)

```
getPendingRepliesAction(): PendingReply[]
```

- 현재 세션 `recipient_id` 기준
- 조건: `sent_at > now()` (아직 도착 전)
- 반환 필드: `id`, `confessionId`, `tone`, `sentAt`, `remainingSeconds` (서버 계산)
- 인증: 기존 `getSession()` 패턴 동일

#### 4.2.2 고해 제출 성공 시 클라이언트 상태

- `submitConfessionAction` 성공 직후, **가장 최근 pending reply** 정보를 클라이언트 상태에 저장
- 본당(CATHEDRAL) 뷰 상단 또는 하단에 **"답장 도착까지 MM:SS"** 카운트다운 배너 표시
- 카운트다운은 `sentAt` 기준 클라이언트 타이머 (1초 간격). 서버 시각과 ±30초 오차 허용 (NFR §7.3)

#### 4.2.3 도착 감지 및 알림

- 카운트다운 `0` 도달 시:
  1. `getUserRepliesAction()` 1회 호출 (즉시 fetch)
  2. 신규 도착 편지가 있으면:
     - 전역 토스트: *"봉인된 편지봉투가 도착했습니다"*
     - 하단 네비 '서신' 탭에 **미읽음 뱃지** (숫자 또는 점)
- 백그라운드 폴링 전략:
  - pending reply 존재 시: **30초** 간격으로 `getUserRepliesAction` 호출
  - pending 없음: 기존 15초 전역 폴링 유지 또는 60초로 완화 (구현 재량, 배터리 우선)

#### 4.2.4 편지함 UI 상태 분기

| 상태 | 조건 | UI |
|---|---|---|
| 대기 중 (봉인) | `sent_at > now` | 봉투 실루엣 + 자물쇠 아이콘 + "도착까지 MM:SS" + 톤 뱃지(천사/악마) |
| 도착·미읽음 | `sent_at <= now && !isRead` | 봉투 글로우 + "터치하여 개봉" CTA |
| 도착·읽음 | `isRead === true` | 기존 편지 카드 (현행과 동일) |

#### 4.2.5 읽음 처리 (Server Action 신규)

```
markReplyAsReadAction(replyId: string): { success: boolean }
```

- 본인 `recipient_id` 소유 reply만 `is_read = true` 업데이트
- 개봉(읽음) 시 미읽음 뱃지 카운트 감소

#### 4.2.6 빈 상태 카피 수정

- 기존 *"새로고침 버튼을 터치"* 문구 **삭제**
- 대체: *"고해를 태우면 이곳에 봉인된 편지가 먼저 나타납니다. 5분 뒤 자동으로 개봉할 수 있습니다."*

## 6. 데이터 모델 / Supabase 스키마 영향

### 6.1 스키마 변경: **없음**

기존 `replies` 테이블 필드로 충분하다.

| 컬럼 | 활용 |
|---|---|
| `sent_at` | 도착 예정 시각 / 도착 여부 판별 |
| `is_read` | 읽음 처리 (현재 insert 시 `false`, update만 추가) |
| `recipient_id` | 세션별 필터 |
| `tone` | 봉인 상태 뱃지 표시 |

### 6.2 쿼리 추가 (애플리케이션 레벨)

```sql
-- 대기 중 (pending)
SELECT * FROM replies
WHERE recipient_id = :userId AND sent_at > now()
ORDER BY sent_at ASC;

-- 도착·미읽음 (unread count)
SELECT COUNT(*) FROM replies
WHERE recipient_id = :userId AND sent_at <= now() AND is_read = false;
```

### 6.3 RLS / 권한

- 기존 anon 클라이언트 패턴 유지
- `markReplyAsRead`는 `recipient_id` 일치 검증 필수 (Server Action에서 세션 대조)

## 7. 디자인 요청 사항

구현 전 디자인 단계에 전달할 항목이다.

### 7.1 카운트다운 배너 (본당 피드 상단)

- 모바일 퍼스트, 한 줄~두 줄 컴팩트 배너
- 시각 요소: 작은 시계/모래시계 아이콘 + `MM:SS` monospace 숫자
- 톤: 기존 amber/cyan 다크 고딕 톤 유지
- 카운트다운 종료 시 배너가 **페이드아웃** 후 토스트로 전환

### 7.2 편지함 — 봉인 봉투 카드

- 기존 편지 카드와 동일한 `rounded-3xl` 스케일
- 봉인 상태: 실루엣만 보이고 본문 블러/마스킹, 자물쇠(Lock) 아이콘
- 톤별 테두리 글로우 (천사=cyan, 악마=rose)
- 도착 후: 봉투 봉인선이 열리는 **짧은 모션** (0.6~1.0초, `prefers-reduced-motion` 시 즉시 전환)

### 7.3 네비게이션 미읽음 뱃지

- '서신' 아이콘 우상단 8px 원형 뱃지
- 1건: 점(dot), 2건 이상: 숫자 (최대 "9+")
- amber-500 계열, 기존 네비와 대비 확보

### 7.4 도착 토스트

- 기존 success 토스트 스타일 재사용 (```363:373:src/app/page.tsx```)
- 문구 예: *"성찰의 시간이 끝났습니다. 편지봉투가 도착했어요."*
- 탭 시 편지함(LETTER_BOX)으로 이동 (선택)

- [ ] 카운트다운이 0이 되면 **30초 이내** 편지가 편지함에 나타난다 (서버 `sent_at` 기준)
- [ ] 도착 시 **토스트 알림**이 1회 표시된다 (중복 스팸 없음)
- [ ] 하단 네비 '서신'에 **미읽음 뱃지**가 표시되고, 편지 개봉(읽음) 후 사라진다
- [ ] 편지함에서 `sent_at > now` 인 편지는 **본문이 보이지 않고** 봉인 UI로 표시된다
- [ ] 봉인 편지에 **남은 시간 MM:SS**가 표시된다
- [ ] `sent_at <= now` 이고 미읽음인 편지는 **개봉 CTA**로 열람 가능하다
- [ ] 편지 열람 시 `is_read`가 true로 갱신되고, 재진입 시 읽음 상태가 유지된다
- [ ] 타인 세션의 reply에 `markReplyAsReadAction` 호출 시 **실패**한다
- [ ] 빈 편지함 안내 문구에서 "수동 새로고침" 문구가 **제거**되었다

### 비기능

- [ ] pending 없을 때 폴링이 과도하지 않다 (15~60초 범위, 배터리 고려)
- [ ] 모든 신규 UI 텍스트가 **한국어**이다
- [ ] Server Actions만 사용하며 **신규 API Route를 추가하지 않는다**
- [ ] LLM API를 사용하지 않는다
- [ ] 개인정보를 추가 수집하지 않는다
- [ ] 모바일 360px 뷰포트에서 카운트다운·봉투 카드가 **레이아웃 깨짐 없이** 표시된다
- [ ] `prefers-reduced-motion` 또는 설정의 reduced-motion ON 시 봉투 개봉 모션을 **생략**한다

### 회귀

- [ ] 기존 고해 작성·연소·피드·촛불·스테인드글라스·설정 기능이 정상 동작한다
- [ ] 위기 키워드 포함 고해의 답장에 상담 안내가 **기존과 동일하게** 포함된다
