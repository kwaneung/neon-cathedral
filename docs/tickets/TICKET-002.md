## 목표

네온 성당의 **전 화면 UI를 드리블(Dribbble)에 올릴 수 있는 완성도**로 전면 재디자인한다. 기능·데이터·Server Actions·DB·비즈니스 로직은 **일절 변경하지 않는다**. 순수 비주얼·타이포·모션·토큰 개편만 허용한다.

---

## 개편 대상 화면

| 화면 | 현재 뷰 키 | 비고 |
|---|---|---|
| 성당 입구 (랜딩) | `ENTRANCE` | 히어로·입장 CTA |
| 고해실 (작성) | `CONFESS` | 텍스트에어리어·톤 카드·태우기 |
| 본당 (피드) | `CATHEDRAL` | 고해 카드·촛불·카운트다운 배너 |
| 스테인드글라스 벽 | `STAINED_GLASS` | 모자이크 그리드·라이트박스 |
| 편지함 | `LETTER_BOX` | 봉인/개봉/읽음 카드 |
| 설정 | `SETTINGS` | BGM·모션·세션 안내 |
| 공통 | — | 헤더, 하단 네비, 토스트, `CandleButton`, `BurnEffect`, `LetterCountdownBanner`, `SealedLetterCard` |

---

## 불변 제약

- 모든 기존 기능·플로우·Server Actions·Supabase 스키마 **무변경**
- 모바일 퍼스트 **360px~** 완결, 터치 타겟 **최소 44×44px**
- `prefers-reduced-motion` / 설정 reduced-motion 존중
- 어두운 배경 위 텍스트 **WCAG AA** 이상
- UI 텍스트 **한국어** 유지 (카피 톤만 다듬기 가능, 의미 변경 금지)

---

## 디자인 에이전트 재량

- **아트 디렉션·컬러·타이포·스페이싱·그림자·모션·디자인 토큰 전면 재정의 가능**
- 유지할 컨셉만 고정: **"네온 성당"** — 어두운 성당 공간 + 네온/촛불의 온기
- 컴포넌트 분리·CSS 토큰화는 허용하되, props/액션 시그니처와 데이터 바인딩은 유지

---

## 완료 판정

1. 위 전 화면·공통 요소에 리디자인이 적용되어 있다.
2. 입장→고해→피드→촛불→벽화→편지함→설정 플로우에 **기능 회귀가 없다**.
3. `lint` / `build`가 통과한다.
4. 360px 모바일에서 레이아웃이 깨지지 않는다.
5. reduced-motion ON 시 과도한 모션이 억제된다.
